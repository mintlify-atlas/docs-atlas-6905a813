---
title: "Motion settings"
description: "Configure steps per mm, feed rates, acceleration, jerk, S-curve acceleration, and Linear Advance in Marlin."
---

Motion settings live primarily in the `// @section motion` block of `Configuration.h` (line 1316) and the `// @section extruder` block of `Configuration_adv.h`. All of the values described here can be changed at runtime with G-code and optionally saved to EEPROM with `M500`.

<Note>
Settings saved to EEPROM override the defaults in `Configuration.h`. If you change a value in the firmware and it doesn't seem to take effect, run `M502` to restore factory defaults, then `M500` to save.
</Note>

## Steps per unit

`DEFAULT_AXIS_STEPS_PER_UNIT` is the most critical motion setting. It tells Marlin how many stepper motor steps correspond to one millimeter of linear movement (or one degree for rotational axes).

```cpp
/**
 * Default Axis Steps Per Unit (linear=steps/mm, rotational=steps/°)
 * Override with M92 (when enabled below)
 *                                      X, Y, Z [, I [, J [, K...]]], E0 [, E1[, E2...]]
 */
#define DEFAULT_AXIS_STEPS_PER_UNIT   { 80, 80, 400, 500 }
```

The array order is `{ X, Y, Z, E0 }`. For a printer with additional axes (I, J, K, U, V, W) or multiple extruders, extend the array accordingly.

**Common values:**

| Axis | Typical value | Notes |
|---|---|---|
| X, Y | `80` | 20-tooth GT2 pulley, 1/16 microstepping, 200-step motor |
| X, Y | `100` | 16-tooth GT2 pulley, 1/16 microstepping |
| Z | `400` | M5 leadscrew (0.8 mm pitch), 1/16 microstepping |
| Z | `800` | M5 leadscrew, 1/32 microstepping |
| E | `93` | Bondtech BMG or similar with 3:1 gear reduction |
| E | `415` | Prusa i3 MK3 extruder |

Use the [Průša calculator](https://blog.prusa3d.com/calculator_3416/) or the RepRap calibration guides to compute the correct values for your specific hardware. Override at runtime with `M92`.

Enable `EDITABLE_STEPS_PER_UNIT` (on by default) to allow `M92` to change steps/mm at runtime:

```cpp
// Enable support for M92. Disable to save at least ~530 bytes of flash.
#define EDITABLE_STEPS_PER_UNIT
```

## Maximum feed rate

`DEFAULT_MAX_FEEDRATE` sets the top speed for each axis in mm/s. The printer will never exceed these values regardless of what a slicer requests.

```cpp
/**
 * Default Max Feed Rate (linear=mm/s, rotational=°/s)
 * Override with M203
 *                                      X, Y, Z [, I [, J [, K...]]], E0 [, E1[, E2...]]
 */
#define DEFAULT_MAX_FEEDRATE          { 300, 300, 5, 25 }
```

The default values (`{ 300, 300, 5, 25 }`) are conservative. X and Y at 300 mm/s is safe for most Cartesian printers; the Z axis at 5 mm/s is appropriate for a standard leadscrew. The extruder limit of 25 mm/s prevents the motor from stalling during fast retract moves.

Override at runtime with `M203`. Example: `M203 X500 Y500` to allow faster XY travel on a CoreXY.

## Maximum acceleration

`DEFAULT_MAX_ACCELERATION` limits how quickly each axis can change speed (in mm/s²).

```cpp
/**
 * Default Max Acceleration (speed change with time) (linear=mm/(s^2), rotational=°/(s^2))
 * Override with M201
 *                                      X, Y, Z [, I [, J [, K...]]], E0 [, E1[, E2...]]
 */
#define DEFAULT_MAX_ACCELERATION      { 3000, 3000, 100, 10000 }
```

Higher values give faster prints but increase ringing and vibration artifacts. Lower values produce smoother output but slow down the printer. Override at runtime with `M201`.

## Printing, retract, and travel acceleration

These three settings provide more granular control over acceleration for different move types:

```cpp
/**
 * Default Acceleration (speed change with time) (linear=mm/(s^2), rotational=°/(s^2))
 * Override with M204
 *
 *   M204 P    Acceleration
 *   M204 R    Retract Acceleration
 *   M204 T    Travel Acceleration
 */
#define DEFAULT_ACCELERATION          3000    // X, Y, Z and E acceleration for printing moves
#define DEFAULT_RETRACT_ACCELERATION  3000    // E acceleration for retracts
#define DEFAULT_TRAVEL_ACCELERATION   3000    // X, Y, Z acceleration for travel (non printing) moves
```

| Setting | G-code | Purpose |
|---|---|---|
| `DEFAULT_ACCELERATION` | `M204 P` | Used during printing moves (XYZ + E together) |
| `DEFAULT_RETRACT_ACCELERATION` | `M204 R` | Used for extruder-only retract/recover moves |
| `DEFAULT_TRAVEL_ACCELERATION` | `M204 T` | Used for travel moves (no extrusion) |

Travel acceleration can safely be set much higher than print acceleration since there's no filament being pushed.

## Jerk control

Marlin offers two approaches to handling instantaneous speed changes at corners: Classic Jerk and Junction Deviation.

<Tabs>
  <Tab title="Junction Deviation (default)">
    Junction Deviation is enabled by default when `CLASSIC_JERK` is not defined. It uses a physics-based model to compute the maximum cornering speed dynamically, generally producing better quality than classic jerk.

    ```cpp
    /**
     * Junction Deviation Factor
     * See: https://reprap.org/forum/read.php?1,739819
     */
    #if DISABLED(CLASSIC_JERK)
      #define JUNCTION_DEVIATION_MM 0.013 // (mm) Distance from real junction edge
      #define JD_HANDLE_SMALL_SEGMENTS    // Use curvature estimation for small segments
    #endif
    ```

    `JUNCTION_DEVIATION_MM` is the key tuning value. Smaller values produce rounder corners (more conservative), larger values allow faster cornering. A starting value of `0.013` mm is reasonable for most printers. Override at runtime with `M205 J`.
  </Tab>
  <Tab title="Classic Jerk">
    Enable `CLASSIC_JERK` to use per-axis jerk limits instead. Jerk specifies the minimum speed change that requires acceleration — changes smaller than the jerk limit happen instantaneously.

    ```cpp
    //#define CLASSIC_JERK
    #if ENABLED(CLASSIC_JERK)
      #define DEFAULT_XJERK 10.0
      #define DEFAULT_YJERK 10.0
      #define DEFAULT_ZJERK  0.3
      #define DEFAULT_EJERK  5.0
    #endif
    ```

    Lower jerk values produce smoother corners but slow the printer. Higher values allow faster corner speeds but can cause ringing. Override at runtime with `M205 X Y Z E`. Classic Jerk is simpler to tune for delta printers.
  </Tab>
</Tabs>

## S-curve acceleration

`S_CURVE_ACCELERATION` replaces the default trapezoidal acceleration profile with a Bézier curve, smoothing the transition at the start and end of acceleration phases. This reduces vibration and ringing without requiring input shaping hardware.

```cpp
/**
 * S-Curve Acceleration
 *
 * This option eliminates vibration during printing by fitting a Bézier
 * curve to move acceleration, producing much smoother direction changes.
 */
//#define S_CURVE_ACCELERATION
```

Uncomment to enable. `S_CURVE_ACCELERATION` is incompatible with `LIN_ADVANCE` in versions prior to Marlin 2.1 — check the `Configuration_adv.h` comments for your version.

## Linear Advance

Linear Advance (LA) compensates for the elasticity of the filament path. When the extruder accelerates, it pre-pushes extra filament to prevent under-extrusion at the start of a move; when it decelerates, it retracts early to prevent ooze.

Enable it in `Configuration_adv.h`:

```cpp
// @section extruder

/**
 * Linear Pressure Control v1.5
 *
 * K=0 means advance disabled.
 * Set K around 0.22 for 3mm PLA Direct Drive with ~6.5cm between the drive gear and heatbreak.
 *
 * See https://marlinfw.org/docs/features/lin_advance.html for full instructions.
 */
//#define LIN_ADVANCE

#if ANY(LIN_ADVANCE, FT_MOTION)
  #define ADVANCE_K 0.22  // (mm) Compression length for all extruders. Override with 'M900 K<mm>'.
#endif
```

The `ADVANCE_K` value (called K factor) must be tuned for your specific printer, hotend, and filament type. Use the [Marlin Linear Advance calibration print](https://marlinfw.org/docs/features/lin_advance.html) to find the right value.

Common starting points:
- Direct drive with short filament path: `0.05` – `0.12`
- Bowden tube: `0.4` – `2.0` (depending on tube length and filament)

Override at runtime with `M900 K<value>`. Save with `M500`.

<Warning>
`LIN_ADVANCE` requires that `DEFAULT_MAX_ACCELERATION` for the E axis is set high enough (10000 mm/s² or more is typical). If the required E acceleration exceeds this limit, Marlin will automatically reduce print acceleration during affected moves.
</Warning>

## Input Shaping

For printers where ringing is a problem, Marlin supports Zero Vibration (ZV) Input Shaping in `Configuration_adv.h`. This approach measures the resonant frequency of each axis and applies a filter to cancel the vibration.

```cpp
// @section motion control

//#define INPUT_SHAPING_X
//#define INPUT_SHAPING_Y
#if ANY(INPUT_SHAPING_X, INPUT_SHAPING_Y, INPUT_SHAPING_Z)
  #if ENABLED(INPUT_SHAPING_X)
    #define SHAPING_FREQ_X  40.0   // (Hz) Dominant resonant frequency on X axis.
    #define SHAPING_ZETA_X   0.15  // Damping ratio (0.0 = no damping, 1.0 = critical)
  #endif
  #if ENABLED(INPUT_SHAPING_Y)
    #define SHAPING_FREQ_Y  40.0
    #define SHAPING_ZETA_Y   0.15
  #endif
#endif
```

Tune with `M593 D<damping> F<frequency>`. Measure your printer's resonant frequency with an accelerometer or by printing a ringing test object.
