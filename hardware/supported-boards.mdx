Marlin 2.1 runs on both 8-bit AVR and 32-bit ARM hardware. A single shared codebase targets all platforms through the [Hardware Abstraction Layer (HAL)](/hardware/hal). The `MOTHERBOARD` define in `Configuration.h` selects the correct pin file and platform-specific code at compile time.

## Platform table

The following platforms are supported. Each entry corresponds to a HAL directory under `Marlin/src/HAL/` and a matching set of board definitions in `Marlin/src/core/boards.h`.

| Platform | MCU | Example boards |
|---|---|---|
| Arduino AVR | ATmega (1280, 2560, 644P, 1284P, AT90USB1286) | RAMPS, Melzi, RAMBo |
| Teensy++ 2.0 | AT90USB1286 | Printrboard |
| Arduino Due | SAM3X8E (ARM Cortex-M3) | RAMPS-FD, RADDS, RAMPS4DUE |
| ESP32 | Xtensa LX6 | FYSETC E4, E4d@BOX, MRR ESPA/ESPE |
| GD32 | GD32 ARM Cortex-M3/M4 | Creality MFL GD32 V4.2.2, Voxelab Aquila (GD32) |
| HC32 | HC32 ARM Cortex-M4 | Ender-2 Pro, Voxelab Aquila |
| LPC1768 | ARM Cortex-M3 | MKS SBASE, Re-ARM, Selena Compact, BTT SKR V1.x |
| LPC1769 | ARM Cortex-M3 | Smoothieboard, Azteeg X5 Mini, TH3D EZBoard, BTT SKR V1.4 TURBO |
| Pico RP2040 | Dual ARM Cortex-M0+ | Raspberry Pi Pico, BTT SKR Pico |
| STM32F103 | ARM Cortex-M3 | Malyan M200, GTM32 Pro, MKS Robin, BTT SKR Mini |
| STM32F401 | ARM Cortex-M4 | Artillery Ruby, FYSETC Cheetah V2/V3, BTT SKR Mini E3 V3.0.1 |
| STM32F4xx | ARM Cortex-M4 | ARMED, Rumba32, BTT SKR Pro, Lerdge, FYSETC S6, BTT Octopus |
| STM32F7x6 | ARM Cortex-M7 | RemRam V1 |
| STM32G0B1 | ARM Cortex-M0+ | BTT SKR Mini E3 V3.0, BTT Manta M4P/M5P/M8P, BTT SKRat |
| STM32H743 | ARM Cortex-M7 | BTT SKR V3.0, BTT SKR SE BX V2/V3, BTT Octopus Pro V1.1 |
| SAMD21 | ARM Cortex-M0+ | Minitronics v2.0 |
| SAMD51 | ARM Cortex-M4 | Adafruit Grand Central M4, RAMPS 1.4.4 |

| Teensy 3.5/3.6 | MK64FX512 / MK66FX1M0 ARM Cortex-M4 | — |
| Teensy 4.0/4.1 | MIMXRT1062 ARM Cortex-M7 | T41U5XBB |
| Linux Native | x86 / ARM / RISC-V | Raspberry Pi GPIO |
| Simulator | Windows, macOS, Linux | Desktop OS |

<Note>
The Simulator (`BOARD_SIMULATED`, board number 9999) lets you run Marlin on a desktop OS without physical hardware. It is intended for UI development and automated testing, not for controlling a real printer. See the `NATIVE_SIM` HAL directory for implementation details.
</Note>

## 8-bit AVR boards

Marlin began on Arduino-based 8-bit hardware and continues to support it. These boards are defined in `boards.h` with IDs in the `1000–1999` range.

### RAMPS 1.3 / 1.4 (ATmega1280, ATmega2560)

The most common AVR platform. Pin assignments live in `src/pins/ramps/`.

| Board constant | Board ID | Description |
|---|---|---|
| `BOARD_RAMPS_14_EFB` | 1020 | RAMPS 1.4 — Hotend, Fan, Bed |
| `BOARD_RAMPS_14_EEB` | 1021 | RAMPS 1.4 — Hotend0, Hotend1, Bed |
| `BOARD_RAMPS_14_EFF` | 1022 | RAMPS 1.4 — Hotend, Fan0, Fan1 |
| `BOARD_RAMPS_14_EEF` | 1023 | RAMPS 1.4 — Hotend0, Hotend1, Fan |
| `BOARD_RAMPS_BTT_16_PLUS_EFB` | 1040 | BigTreeTech RAMPS 1.6+ — Hotend, Fan, Bed |
| `BOARD_MKS_BASE` | 1107 | MKS BASE v1.0 |
| `BOARD_MKS_GEN_L` | 1113 | MKS GEN L |
| `BOARD_RUMBA` | 1127 | Rumba |
| `BOARD_ULTIMAKER` | 1122 | Ultimaker |
| `BOARD_RAMPS_CREALITY` | 1140 | Creality CR10S, CR20, CR-X |
