---
title: "LCD displays & controllers"
description: "Supported display types in Marlin and how to select and configure them in Configuration.h."
---

Marlin supports a wide range of display hardware — from simple character LCDs to full-color touchscreen TFTs. The display is configured by enabling exactly one controller define in `Configuration.h`. The LCD section is divided into several groups: character-based LCDs, I2C and shift-register LCDs, graphical DOGM LCDs, OLED displays, extensible-UI (touch) displays, graphical TFTs, and DWIN-based displays used on Creality machines.

<Note>
Enable only one display type at a time. Enabling multiple controller defines simultaneously will cause compilation errors or unpredictable behavior.
</Note>

## Character-based LCDs (HD44780)

Character LCDs use the HD44780 (or compatible) controller and display text in a fixed grid of characters. They are inexpensive and do not require a graphics library.

The character set is selected with:

```cpp
// Configuration.h
#define DISPLAY_CHARSET_HD44780 JAPANESE  // JAPANESE, WESTERN, or CYRILLIC
```

Common character LCD options:

```cpp
// Configuration.h — Character-based LCDs section

// RepRapDiscount Smart Controller (20×4, white PCB, SD card, rotary encoder)
//#define REPRAP_DISCOUNT_SMART_CONTROLLER

// Ultimaker Controller
//#define ULTIMAKERCONTROLLER

// ULTIPANEL (generic 20×4 with encoder, as seen on Thingiverse)
//#define ULTIPANEL

// ANET and Tronxy 20×4 LCD2004 with 5 analog buttons
//#define ZONESTAR_LCD

// Generic 16×2, 16×4, 20×2, or 20×4 character LCD
//#define ULTRA_LCD
```

## Graphical LCDs (DOGM — 128×64)

Graphical LCDs render a full 128×64 pixel bitmap. Marlin uses the U8glib library for these displays. If your graphical LCD is unresponsive, try reversing the EXP1/EXP2 connector plugs.

```cpp
// Configuration.h — Graphical LCDs section

// RepRapDiscount FULL GRAPHIC Smart Controller (128×64, most common graphical controller)
//#define REPRAP_DISCOUNT_FULL_GRAPHIC_SMART_CONTROLLER

// Creality CR-10 / CR-7 / Ender-3 stock display (128×64, connects to EXP1)
//#define CR10_STOCKDISPLAY

// Ender-2 stock display (variant of MKS MINI 12864)
//#define ENDER2_STOCKDISPLAY

// MKS MINI 12864 with graphic controller and SD support
//#define MKS_MINI_12864

// BigTreeTech Mini 12864 V1.0 / V2.0 (NeoPixel RGB backlight)
//#define BTT_MINI_12864

// FYSETC Mini 12864 variants
//#define FYSETC_MINI_12864_X_X    // Type C/D/E/F — no tunable RGB backlight
//#define FYSETC_MINI_12864_1_2    // Type C/D/E/F — simple RGB backlight
//#define FYSETC_MINI_12864_2_0    // Type A/B — discreet RGB backlight
//#define FYSETC_MINI_12864_2_1    // Type A/B — NeoPixel RGB backlight
```

## OLED displays

OLED displays use SSD1306 or SH1106 controllers and produce very sharp output at small sizes:

```cpp

//#define U8GLIB_SSD1306

// SAV 3D OLED with SSD1306 or SH1106
//#define SAV_3DGLCD
#if ENABLED(SAV_3DGLCD)
  #define U8GLIB_SSD1306
  //#define U8GLIB_SH1106
#endif

// MKS OLED 1.3" 128×64
```

## DGUS touch displays

DGUS displays from DWIN run their own OS and communicate with Marlin over serial. Different UI styles require different resources to be copied to the display's SD card.

```cpp
// Configuration.h
// :[ 'ORIGIN', 'FYSETC', 'HYPRECY', 'MKS', 'RELOADED', 'IA_CREALITY', 'E3S1PRO' ]
//#define DGUS_LCD_UI ORIGIN
```

Each variant requires a corresponding resource package flashed to the display itself. See the comment block in `Configuration.h` for download links for each style.

## FTDI EVE touch displays

The FTDI EVE (FT800/FT810) touch UI is enabled with:

```cpp
// Configuration.h
//#define TOUCH_UI_FTDI_EVE
```

Further configuration (display resolution, SPI pins, orientation) is done in `Configuration_adv.h`.

## Graphical TFTs

Graphical TFT displays use FSMC or SPI interfaces and are common on Creality, MKS, and BigTreeTech boards. Three UI styles are available for these displays:

```cpp
// Configuration.h — TFT UI selection (choose one)
//#define TFT_CLASSIC_UI  // Emulated DOGM 128×64 upscaled — familiar Marlin look
//#define TFT_COLOR_UI    // Full-color Marlin menus with touch support
//#define TFT_LVGL_UI     // Modern UI using the LVGL graphics library
```

Enable touch input for TFT displays with:

```cpp
//#define TOUCH_SCREEN
#if ENABLED(TOUCH_SCREEN)
  #define BUTTON_DELAY_EDIT      50 // (ms) repeat delay for edit screens
  #define BUTTON_DELAY_MENU     250 // (ms) repeat delay for menus
  #define TOUCH_SCREEN_CALIBRATION
#endif
```

## DWIN displays (Creality Ender-3 V2 and variants)

The DWIN display used on the Ender-3 V2 and similar Creality machines connects over UART and has its own internal processor. Several UI options are available:

```cpp
// Configuration.h
//#define DWIN_CREALITY_LCD           // Stock Creality UI
//#define DWIN_LCD_PROUI              // Pro UI by MRiscoC
//#define DWIN_CREALITY_LCD_JYERSUI   // Jyers UI by Jacob Myers
//#define DWIN_MARLINUI_PORTRAIT      // MarlinUI (portrait orientation)
//#define DWIN_MARLINUI_LANDSCAPE     // MarlinUI (landscape orientation)
```

## Display orientation

For TFT displays, set the rotation with:

```cpp
// Configuration.h
// Options: TFT_NO_ROTATION, TFT_ROTATE_90, TFT_ROTATE_180, TFT_ROTATE_270

//#define TFT_ROTATION TFT_NO_ROTATION
```
